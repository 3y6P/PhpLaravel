Основы отзывчивого веб­‑дизайна

Отзывчивый веб­‑дизайн (англ. responsive web design) ⁠— это подход, позволяющий веб­‑странице приспосабливаться к размерам области просмотра на дисплеях разнотипных устройств: мониторов, планшетов, смартфонов и т. п. Следует различать отзывчивый и адаптивный веб­‑дизайн: первый опирается на концепцию «один шаблон для бесконечного множества устройств» и медиа­‑запросы CSS, второй ⁠— на концепцию «по одному шаблону для каждого подмножества устройств». Классический пример адаптивной вёрстки ⁠— демонстрационный сайт Кэмерона Адамса, где сценарий JavaScript в зависимости от ширины области просмотра активирует ту или иную таблицу стилей.

Отзывчивый веб­‑дизайн подчинён стратегии поступательного совершенствования (англ. progressive enhancement), которая требует обеспечить доступность основного контента веб­‑страницы во всех пользовательских агентах (браузерах), в т. ч. текстовых, и принуждает наращивать возможности веб­‑страниц таким образом, чтобы работоспособность более простых техник не зависела от более продвинутых. Так, отключение JavaScript не должно сказаться на оформлении или привести к полной неработоспособности элементов управления: меню, кнопок и пр. (эта концепция называется unobtrusive JavaScript ⁠— «ненавязчивый JavaScript»), отсутствие поддержки CSS ⁠— на читабельности текста и т. п.

Сохраните следующий фрагмент разметки как текстовый файл в кодировке UTF‑8 без BOM. 
<!doctype html>
<html lang="ru">
  <head>
    <meta charset="utf-8">
    <title>Заготовка для шаблона Bootstrap</title>
  </head>
  <body>
  </body>
</html>

Рекомендация HTML 5.1 требует размещать декларацию кодировки символов в составе последовательности из первых 1024 байтов документа [@html-5.1, § 4.2.5.5]. Для обратной совместимости со старыми браузерами требования ужесточаются: так, Internet Explorer 7 искал декларацию среди первых 256 октетов.

В целях безопасности размещайте декларацию до любого контента, особенно динамического, в т. ч. извлечённого из базы данных.

Метаданные

Вставьте следующий фрагмент разметки между тегами <head> и </head>, желательно до стартового тега <title>. 
<meta charset="utf-8">
<meta http-equiv="X-UA-Compatible" content="IE=Edge">
<meta name="HandheldFriendly" content="true">
<meta name="MobileOptimized" content="0">
<meta name="viewport"
      content="width=device-width, initial-scale=1, shrink-to-fit=no">

Ключ HandheldFriendly появился в браузере AvantGo для карманных персональных компьютеров и для смартфонов BlackBerry. Значение true декларирует совместимость вёрстки веб­‑страницы с КПК, планшетам и смартфонам.

Ключ MobileOptimized впервые реализовали в 2005 г. в браузере Pocket Internet Explorer под управлением операционной системы Windows Mobile 5.0 для КПК PocketPC [@firtman.mobile-web, с. 214]. Ключ позволял верстальщикам указать предпочтительную ширину экрана, причём значение 0 соответствовало ширине экрана на произвольном устройстве [@grimes.ie-mobile-6].

Ключ viewport впервые реализовали в браузере Safari. Среди свойств viewport следует выделить width, initial-scale и user-scalable.

Свойство width со значением device-width декларировало, что ширина веб­‑страницы адаптируется под ширину устройства, чем провоцировало побочные эффекты. Так, iPad 3 (2012 г.) даже в ландшафтной ориентации интерпретирует width=device-width как width=768 и для исправления ошибки требует указать initial-scale=1. Safari 9.0 (2015 г.) при установке width=device-width уменьшает масштаб отображения веб­‑страницы, чтобы уместить в области просмотра негабаритные элементы. Прежнее поведение восстанавливает свойство shrink-to-fit=no [@new-in-safari-9.0].

Свойство initial-scale=1 устанавливает единичный коэффициент масштабирования веб­‑страницы при первом отображении [@safari-meta].

Свойство user-scalable с установленным по умолчанию значением yes разрешает пользователю масштабировать веб­‑страницу в пределах от minimum-scale до maximum-scale [@safari-meta].

Порядок следования ключей неслучаен и следует стратегии progressive enhancement, так как MobileOptimized переопределяет HandheldFriendly, а viewport переопределяет и MobileOptimized, и HandheldFriendly [@viewport-on-windows-phone-7].

Опытные веб­‑разработчики размещают таблицы стилей CSS, сценарии JavaScript и другие утилитарные ресурсы в сетях доставки контента ⁠— content delivery network, сокращённо CDN. Сайты, которые ссылаются на один и тот же ресурс в одной и той же CDN, разрешают пользовательскому агенту ⁠— браузеру однократно скачать, а затем кешировать содержимое ресурса на установленный CDN срок и при последующих запросах по тому же URL заменить скачивание извлечением из кеша, с локального носителя.

Однако злоумышленник способен изменить содержимое ресурса на сервере или вовсе перенаправить запрос на другой узел, причём TLS не пресечёт атаку, поскольку гарантирует подлинность узла, но не достоверность контента, а подложный узел наверняка предъявит годный сертификат. В этой связи Консорциум Всемирной паутины издал рекомендацию по контролю целостности субресурсов. Согласно спецификации, пользовательский агент отказывает в обработке контента, контрольная сумма которого не совпадает со значения атрибута integrity. Рекомендация 2016 г. и редакторский черновик за 26.08.2017 г. обязывают обеспечить расчёт контрольных сумм посредством хеш­‑функций SHA‑256, SHA‑384, SHA‑512 и распространяются только на элементы HTML link и script ⁠— поддержка a, audio, embed, iframe, img, object, source, track, video ожидается в последующих ревизиях.

Контейнеры

Контейнеры ⁠— базовый элемент вёрстки в Bootstrap.

Адаптивная веб­‑вёрстка опирается на элементы, ширина, высота и положение которых изменяется в зависимости от размеров области просмотра. Старые спецификации, такие как CSS basic box model, слишком скупы на выразительные средства. Новые,

Ширина фиксированного контейнера зависит от ширины родительского элемента.

Ширина родителя, пикселей < 576 ⩾ 576 ⩾ 768 ⩾ 992 ⩾ 1200
Ширина контейнера, пикселей auto 540 720 960 1140

По умолчанию ширина блочного элемента HTML равна ширине родителя, следовательно, элементы nav и article не поместятся в одной строке и расположатся по вертикали. Для упрощения будем считать, что ширина div.row равна ширине области просмотра.

<nav     class="bg-primary">nav</nav>
<article class="bg-warning">article</article>

Поместим в контейнер Bootstrap.

<div class="container-fluid">
  <div class="row">
    <nav     class="bg-primary">nav</nav>
    <article class="bg-warning">article</article>
  </div>
</div>

Контейнер сжал блоки до минимума, по содержимому, и разместил в строку! Вёрстка выглядит неряшливо, но ведь мы не довели задуманное до конца ⁠— не задали ширину блоков посредством классов с префиксом col-, которые, во­‑первых, вернут блокам стопроцентную ширину по умолчанию, а во­‑вторых, зададут пороговые значения точек останова ⁠— при их достижении ширина, высота и положение блоков будут меняться. Ознакомьтесь с таблицей, чтобы понять, какие размерности заложены в сетку Bootstrap.

Префикс класса .col- .col-sm- .col-md- .col-lg- .col-xl-
Ширина, пикселей < 576 ⩾ 576 ⩾ 768 ⩾ 992 ⩾ 1200

Рассмотрим пример. Пусть ширина div.row равна или превышает 768 пикселей. Тогда причисление nav и article к классу col-md-6 приравняет ширину каждого из них к 6⁄12 ширины родителя, выровняет по высоте и разместит эти элементы на одной горизонтальной оси. Ширина div.row меньше 768 пикселей вернёт nav и article к стопроцентной ширине и размещению на одной вертикальной оси.

<div class="container-fluid">
  <div class="row">
    <nav     class="col-lg-6 bg-primary">nav</nav>
    <article class="col-lg-6 bg-warning">article</article>
  </div>
</div>

Рассмотрим ещё один пример. Меню обычно занимает меньше места, чем статья, поэтому пусть при средней и большой ширине div.row элемент nav занимает 3⁄12, article ⁠— 9⁄12 ширины родителя. Если ширина div.row равна ширине экрана, то при портретной ориентации iPhone 6/6S/7 элементы nav и article расположатся как две строки, а при ландшафтной ⁠— как два столбца.

<div class="container-fluid">
  <div class="row">
    <nav     class="col-lg-3 bg-primary">nav</nav>
    <article class="col-lg-9 bg-warning">article</article>
  </div>
</div>

Таблицы

Немного терминологии:

  • table;
    • caption ⁠— тематический заголовок таблицы, совокупно с нумерационным заголовком таблицы (CSS) образующий её шапку;
    • thead ⁠— заголовочная часть таблицы, или головка;
      • tr ⁠— ярус;
        • td ⁠— заголовок боковика;
        • th ⁠— заголовок графы;
    • tbody ⁠— хвостовая часть таблицы, или хвост;
      • tr ⁠— строка таблицы;
        • th ⁠— боковик;
        • td ⁠— графа;
    • tfoot ⁠— 

Таблицы, особенно многоколоночные, опасны

, так как не уместятся в области просмотра, и на веб­‑странице появится горизонтальная полоса прокрутки. Класс table-responsive

На текущий момент (август 2017 г.) в инструкции к Bootstrap 4.0.0­‑beta допущена ошибка: класс table-responsive следует назначать не таблице, а её родительскому элементу.

К элементу table можно применить следующие классы:

  • table ⁠— стопроцентная ширина таблицы относительно ширины родительского элемента;
  • table-bordered ⁠— разлиновка граф таблицы;
  • table-inverse ⁠— инверсия цветового оформления таблицы: установка тёмного цвета фона и светлого цвета текста;
  • table-hover ⁠— подсветка графы, на которую наведён указатель мыши;
  • table-striped ⁠— zebra striping. Согласно исследованию 2008 г., построчное чередование фонового цвета граф ускоряет поиск по таблице, нравится большинству ⁠— 31 % и совершенно не нравится только 4 % респондентов. Для сравнения, разлиновку граф предпочли 20 % и отвергли также 4 % участников опроса.

К элементу thead можно применить следующие классы:

  • thead-inverse ⁠— инверсия цветового оформления; эффект заметен, если для таблицы не установлены правила table-inverse.

К любым табличным элементам можно применять следующие классы:

  • table-active, table-danger, table-dark, table-info, table-light, table-primary, table-secondary, table-success, table-warning ⁠— цвет фона с учётом table-hover;
  • bg-danger, bg-info, bg-primary, bg-success, bg-warning ⁠— цвет фона, игнорирующий table-hover;
  • text-danger, text-dark, text-info, text-light, text-primary, text-secondary, text-success, text-warning ⁠— цвет текста.
